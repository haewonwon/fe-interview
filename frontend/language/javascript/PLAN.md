# JavaScript Deep Dive 핵심 스터디 계획

## 1. 스터디 목표

『모던 자바스크립트 Deep Dive』를 처음부터 끝까지 읽는 대신, 자바스크립트의 언어적 특성과 코드의 실행 원리를 설명하는 데 필요한 핵심 장을 선별하여 학습한다.

스터디 종료 시 다음 질문에 자신의 말로 답할 수 있는 상태를 목표로 한다.

- 자바스크립트가 동적 타입 언어라는 것은 무엇을 의미하는가?
- 원시 값과 객체를 할당하거나 전달할 때 어떤 차이가 있는가?
- `var`, `let`, `const`가 모두 호이스팅되는데 동작이 다른 이유는 무엇인가?
- 자바스크립트 엔진은 변수와 함수를 어떻게 관리하고 탐색하는가?
- 클로저가 외부 함수의 변수를 기억할 수 있는 이유는 무엇인가?
- 일반 함수와 화살표 함수의 `this`는 어떻게 결정되는가?
- 객체가 다른 객체의 프로퍼티를 상속받는 원리는 무엇인가?
- 동기 코드, Promise, 타이머 콜백은 어떤 순서로 실행되는가?
- ES Module은 기존 스크립트와 무엇이 다른가?

## 2. 학습 원칙

### 모든 장을 동일하게 읽지 않는다

학습 범위는 다음 세 단계로 구분한다.

- **정독:** 실행 원리를 이해하고 예제를 직접 작성한다.
- **발췌독:** 선정된 절만 읽고 필요한 문법을 확인한다.
- **참고:** 필요할 때 검색하며 이번 스터디에서는 진도에 포함하지 않는다.

### 요약보다 설명과 예측을 우선한다

각 회차에는 반드시 다음 결과물을 남긴다.

1. 핵심 개념을 자신의 말로 설명한 학습 노트
2. 실행 결과 예측 문제 3개와 해설
3. 직접 작성한 최소 예제 1개
4. 실제 프로젝트 코드에서 찾은 관련 사례 또는 적용 아이디어 1개

### 모르는 용어를 그대로 옮겨 적지 않는다

새로운 용어는 다음 형식으로 정리한다.

```text
용어:
한 문장 정의:
왜 필요한가:
관련 코드:
헷갈린 개념과 차이:
```

### 실습

다음 코드의 결과를 실행 전에 설명한다.

```js

function outer() {

  console.log(x);
}

outer();
console.log(x);
```

`var`를 `let`으로 변경했을 때 무엇이 달라지는지도 비교한다.

### 실습

```js

  .then(() => console.log('C'))
  .then(() => console.log('D'));

console.log('E');
```

이후 동일한 비동기 작업을 Promise 체이닝과 `async`/`await` 두 방식으로 작성한다.

6. 프로토타입 체인
7. 이벤트 루프와 마이크로태스크
8. Promise와 `async`/`await`

설명하지 못한 주제는 책 전체를 다시 읽지 않고 관련 절과 작성했던 예제만 복습한다.

### 종합 구현 과제

다음 조건을 만족하는 작은 프로그램을 작성한다.

- 모듈을 2개 이상으로 분리한다.
- 클로저로 내부 상태를 관리한다.
- 배열 고차 함수를 사용한다.
- 객체를 직접 변경하지 않고 새로운 값을 만든다.
- Promise 또는 `async`/`await`로 비동기 작업을 처리한다.
- 에러 발생 시 호출자에게 적절하게 전달한다.

예시 주제:

- 비동기 할 일 목록 저장소
- GitHub 사용자 검색 데이터 가공기
- 영화 목록 필터와 즐겨찾기 저장소
- 프로젝트 일정 통계 계산기

## 8. 이번 스터디에서 제외하는 범위

다음 내용은 중요하지 않다는 뜻이 아니라 이번 목표인 ‘언어의 특성과 주요 실행 원리’에서 우선순위가 낮다는 뜻이다.

- 1~3장: 개발 환경과 역사
- 5장: 표현식과 문의 세부 용어
- 8장: 기본 제어문
- 16장: 프로퍼티 어트리뷰트의 세부 규칙
- 20~21장: strict mode와 빌트인 객체
- 28~33장: Number, Math, Date, 정규 표현식, String, Symbol
- 34.6: 사용자 정의 이터러블과 무한 이터러블
- 38~41장: 브라우저 렌더링, DOM, 이벤트, 타이머
- 43~44장: Ajax와 REST API
- 46.1~46.5: 제너레이터
- 49장: Babel과 Webpack 설정

필요할 때 해당 장을 레퍼런스로 찾아보되 현재 스터디 완료 조건에는 포함하지 않는다.
