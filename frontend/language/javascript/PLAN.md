# JavaScript Deep Dive 핵심 스터디 계획

## 1. 방향

6월에 진행하려던 JavaScript Deep Dive 학습은 7월로 이월한다. 시작일은 **2026년 7월 6일**로 잡는다.

이번 계획은 책을 처음부터 끝까지 읽기 위한 계획이 아니다. 자바스크립트의 언어적 특성과 실행 원리를 설명하는 데 필요한 핵심 주제를 우선순위에 따라 배치한다.

세부 일자별 계획이나 회차별 과제는 잡지 않는다. 7–8월에는 꼭 먼저 잡아야 하는 주제만 보고, 나머지는 9–12월로 넘겨도 된다.

---

## 2. 학습 목표

스터디 종료 시 다음 질문에 자신의 말로 답할 수 있는 상태를 목표로 한다.

- 자바스크립트가 동적 타입 언어라는 것은 무엇을 의미하는가?
- 원시 값과 객체를 할당하거나 전달할 때 어떤 차이가 있는가?
- `var`, `let`, `const`가 모두 호이스팅되는데 동작이 다른 이유는 무엇인가?
- 자바스크립트 엔진은 변수와 함수를 어떻게 관리하고 탐색하는가?
- 클로저가 외부 함수의 변수를 기억할 수 있는 이유는 무엇인가?
- 일반 함수와 화살표 함수의 `this`는 어떻게 결정되는가?
- 객체가 다른 객체의 프로퍼티를 상속받는 원리는 무엇인가?
- 동기 코드, Promise, 타이머 콜백은 어떤 순서로 실행되는가?
- ES Module은 기존 스크립트와 무엇이 다른가?

---

## 3. 운영 원칙

- 시작일은 2026년 7월 6일로 한다.
- 7–8월에는 P1 주제를 우선한다.
- 책의 모든 장을 같은 깊이로 읽지 않는다.
- 예제 결과를 먼저 예측한 뒤 실행한다.
- 요약보다 "왜 그렇게 동작하는지" 설명하는 것을 우선한다.
- TypeScript로 해결할 타입 모델링 주제는 여기서 깊게 다루지 않는다.
- 일정이 밀리면 주제를 삭제하지 않고 9–12월로 이월한다.

## 7. 학습 완료 기준

한 주제를 완료했다고 보려면 다음 중 최소 2개 이상을 남긴다.

- 핵심 개념을 자신의 말로 설명한 학습 노트
- 실행 결과 예측 문제와 해설
- 직접 작성한 최소 예제
- 실제 프로젝트 코드에서 찾은 관련 사례
- 면접 답변 초안

단순히 책을 읽거나 예제를 따라 친 것은 완료로 계산하지 않는다.
