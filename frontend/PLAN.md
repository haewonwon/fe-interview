# 2026년 7–12월 프론트엔드 학습 계획

## 1. 방향

6월에 진행하려던 프론트엔드 학습은 7월로 이월한다. 시작일은 **2026년 7월 6일**로 잡고, 7–8월에는 우선순위가 높은 기반 주제만 배치한다.

이 계획은 UMC에서 했던 내용을 그대로 반복하기 위한 문서가 아니다. 이미 접했던 주제를 바탕으로, 부족했던 기초를 다시 잡고 프로젝트에서 설명할 수 있는 판단 기준을 만드는 것이 목표다.

7–8월에는 프로젝트도 함께 진행하므로 과제 중심으로 빡빡하게 운영하지 않는다. 큰 구현 과제를 새로 만들기보다 작은 실험, 코드 관찰, 프로젝트 적용 후보 정리 위주로 진행한다.

---

## 2. 운영 원칙

- 세부 일자별 계획은 잡지 않는다.
- 한 달에 핵심 주제를 너무 많이 넣지 않는다.
- 7–8월에는 면접과 프로젝트에 바로 연결되는 주제를 우선한다.
- 시간이 부족하면 주제를 삭제하지 않고 9–12월로 넘긴다.
- TypeScript는 별도로 공부할 예정이므로 이 문서에서는 깊게 다루지 않는다.
- UMC 워크북은 참고 자료로만 사용하고, 공식 문서와 실제 코드 동작을 기준으로 보완한다.

- 프론트엔드 테스트
- Vite와 번들링 기본
- GitHub Actions와 배포 자동화
- 성능 측정과 개선

### P3: 11–12월로 넘겨도 되는 주제

## 4. 월별 주제 배치

## 7월: 이월분 정리와 웹 기반

시작일: **2026년 7월 6일**

7월은 6월에 끝내지 못한 내용을 무리해서 몰아넣지 않고, 프론트엔드 기반을 다시 잡는 달로 둔다.

### 공부할 주제

- JavaScript 핵심 실행 원리 이월분
- 시맨틱 HTML
- CSS cascade, box model, Flexbox, Grid
- 접근성 기본
- 브라우저 렌더링 과정
- URL 입력부터 화면 표시까지의 흐름
- DOM, 이벤트, 이벤트 위임
- HTTP 요청과 응답, 상태 코드, CORS

### 남길 것

- 웹 요청과 렌더링 흐름 정리
- HTML, CSS, 접근성 체크리스트
- 이벤트 전파와 이벤트 위임 예제
- `fe-interview` 문서 초안

## 8월: React 핵심과 프로젝트 연결

8월은 프로젝트와 병행하는 달이므로 React 핵심 개념을 프로젝트 코드와 연결하는 데 집중한다.

### 공부할 주제

- React 렌더링과 state
- props, derived state, state 위치 결정
- 불변 업데이트와 `key`
- Effect가 필요한 경우와 필요 없는 경우
- stale closure와 cleanup
- React Router와 URL 상태
- API 요청의 loading, error, empty, success 상태
- 프로젝트 상태 분류
- API 함수와 UI 컴포넌트의 책임 분리

### 남길 것

- 상태를 어디에 둘지 결정하는 기준
- Effect 사용 기준
- URL 상태 적용 후보
- API 에러 처리 기준
- 프로젝트에 적용할 개선 후보

---

## 9월: 서버 상태, 테스트, 빌드

9월은 7–8월에 프로젝트 기반을 잡은 뒤, 데이터 흐름과 품질을 보강하는 달로 둔다.

### 공부할 주제

- Mutation과 캐시 무효화

- Vitest, Testing Library, MSW
- Vite 개발 서버와 프로덕션 빌드
- 환경 변수와 배포 설정

## 6. 학습 완료 기준

한 주제를 완료했다고 보려면 다음 중 최소 2개 이상을 남긴다.

- 자신의 말로 정리한 문서
- 직접 작성한 작은 예제
- 프로젝트 코드에서 찾은 사례
- 선택 기준 또는 트레이드오프 정리
- 면접 답변 초안

단순히 문서를 읽거나 예제를 따라 친 것은 완료로 계산하지 않는다.
