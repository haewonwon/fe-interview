# 그래프

## 그래프란?

- 그래프(graph)란 복잡하게 연결된 객체 사이의 관계를 효율적으로 표현할 수 있는 자료구조임
- 지금까지 공부했던 모든 선형 자료구조나 트리도 그래프로 표현할 수 있음
- 즉, 그래프는 가장 일반화된 자료구조임

### 그래프의 유래

- 강으로 나누어진 네 개의 지역들을 연결하기 위해 7개의 다리를 만듦
- 모든 다리를 한 번씩만 건너서 출발했던 장소로 돌아올 수 있을까? 라는 문제를 오일러가 해결함
- 오일러는 이 문제에서 핵심적이고 중요한 것은 A, B, C, D의 위치가 어떤 관계로 연결되었는가? 라고 생각함
- 따라서 '위치'라는 객체는 정점(vertex)으로, 위치 간의 관계인 '다리'는 간선(edge)으로 표현하여 그래프 문제로 변환함

- 그래프에 존재하는 모든 간선을 한 번만 통과하면서 처음 정점으로 되돌아오는 경로를 오일러 경로(Eulerian tour)라고 정의하고, 그래프의 모든 정점에 연결된 간선의 개수가 짝수일 때만 오일러 경로가 존재한다는 오일러의 정리를 증명함
- 그래프는 정점의 집합과 간선의 집합으로 구성되므로 수학적으로 $G = (V, E)$와 같이 표시됨 ($V$는 정점 집합, $E$는 간선 집합)
- 정점들은 자체로는 큰 의미가 없지만, 이들은 간선으로 연결하면 '관계'가 만들어지고 그래프가 형성됨
- 이때 정점은 객체를 의미하고, 간선은 이러한 객체 간의 관계를 나타내는데, 정점 A와 B를 연결하는 간선은 (A, B)와 같이 정점의 쌍으로 표현함

### 그래프의 종류

그래프는 간선의 방향성과 정점 간의 연결 정도에 따라 여러 가지로 나뉨

1. 무방향 그래프(undirected graph)

- 두 정점을 연결하는 간선에 방향성이 없는 그래프로 간선은 양방향으로 갈 수 있는 길을 의미함
- 두 정점 A와 B를 연결하는 간선은 (A, B)로 표현하는데, (A, B)와 (B, A)는 동일한 간선임

2. 방향 그래프(directed graph)

- 간선에 방향이 있는 그래프로 다이그래프(digraph)라고도 함
- 간선은 한 방향으로만 갈 수 있는 길을 의미하는데, A에 B로 가는 길은 <A, B>로 표현하고 <A, B>와 <B, A>는 서로 다른 간선임

3. 완전(complete) 그래프

- 그래프의 모든 정점 사이에 간선이 존재하는 그래프를 말함
- 정점이 $n$개인 무방향 완전 그래프는 $\dfrac{n(n-1)}{2}$개의 간선을 갖고, 방향 그래프는 $n(n-1)$개의 간선을 가짐

4. 부분 그래프(subgraph)

- 원래의 그래프에서 정점이나 간선 일부만을 이용해 만든 그래프임

5. 가중치 그래프(weighted graph)

- 간선에 가중치가 할당된 그래프를 가중치 그래프 또는 네트워크(network)라고 말함

### 그래프의 용어

- 인접: 간선으로 연결된 두 정점을 인접해 있다고 말함
- 정점의 차수: 그 정점에 연결된 간선의 수를 말함
- 경로: 간선을 따라갈 수 있는 길을 순서대로 나열한 것을 말함
- 단순 경로: 반복되는 간선이 없는 경로를 말함
- 사이클: 시작 정점과 종료 정점이 같은 단순 경로를 말함
- 트리: 사이클을 가지지 않는 연결 그래프를 말함

## 그래프의 표현

- 그래프는 정점과 간선의 집합으로 구성되는데 정점 집합은 리스트를 이용하면 쉽게 표현할 수 있음
- 간선 (u, v)은 정점 u와 v가 인접해 있다는 것을 말하는데, 그래프에서 정점들의 인접 관계를 효율적으로 표현하기 위해 인접 행렬과 인접 리스트를 사용할 수 있음
- 다만, 그래프의 특성과 필요한 연산에 따라 적절한 방법을 선택해야 함

### 인접 행렬을 이용한 표현

- 간선들의 집합을 표현하는 가장 간단한 방법은 2차원 배열을 이용하는 것인데, 이것을 인접 행렬(adjacency matrix)이라 함
- 그래프의 정점 개수가 $n$이라면 인접 행렬(2차원 배열)의 크기는 $n \times n$인데, 행렬의 각 성분이 두 정점의 연결 관계를 나타냄
- 무방향 그래프에서는 인접 행렬이 항상 대칭 행렬임
- 이것은 간선 (U, V)가 U에서 V로의 연결뿐만 아니라 V에서 U로의 연결을 동시에 의미하기 때문임
- 따라서 무방향 그래프는 배열의 상위 삼각이나 하위 삼각만 저장하여 메모리를 절약할 수도 있음

### 인접 리스트를 이용한 표현

- 각 정점이 인접한 정점 리스트를 갖도록 하여 간선들을 표현할 수 있는데, 이러한 리스트를 인접 리스트(adjacency list)라고 함
- 이때 리스트로는 연결 리스트를 사용할 수도 있고, 파이썬의 리스트와 같은 배열 구조를 사용할 수도 있음

### 인접 행렬과 인접 리스트 중에서 어떤 것을 사용할까요?

- 인접 행렬과 인접 리스트 어느 한쪽도 압도적으로 장점이 많지는 않음
- 메모리 사용량이 중요하거나 정점에 비해 간선이 별로 없는 희소 그래프(sparse graph)에서는 인접 리스트가 좋은 선택임
- 정점끼리의 인접 여부를 빨리 알아내야 하거나, 완전 그래프나 이와 유사한 조밀 그래프(dense graph)라면 인접 행렬이 더 좋은 선택이 될 것임

## 그래프 순회

- 그래프 순회는 하나의 정점에서 시작하여 그래프의 모든 정점을 한 번씩 방문하는 작업을 말함
- 그래프의 정점들을 순회하는 체계적인 방법에는 깊이 우선 탐색과 너비 우선 탐색이 있음
- 이들은 이진 트리의 순회 방법과 비교해 볼 수 있음 (트리는 그래프의 일종임)

1. 깊이 우선 탐색(DFS: Depth First Search)

- 시작 정점에서 한 방향으로 계속 가다가 더 이상 갈 수 없으면 가장 가까운 갈림길로 다시 돌아와 다른 방향을 다시 탐색함
- 이것은 이진 트리의 전위순회와 비슷함

2. 너비 우선 탐색(BFS: Breadth First Search)

- 시작 정점에서 가까운 정점을 먼저 방문하고 먼 정점을 나중에 방문함
- 이것은 이진 트리의 레벨순회와 비슷함

### 깊이 우선 탐색

- DFS는 시작 정점에서 한 방향으로 갈 수 있는 곳까지 깊이 탐색을 진행하다가 더 이상 갈 곳이 없으면 가장 최근에 만났던 갈림길 정점으로 되돌아옴
- 갈림길로 돌아와서는 가 보지 않은 다른 방향의 간선으로 탐색을 다시 진행하고, 이 과정을 반복해 결국 모든 정점을 방문함
- 탐색 과정에서 여러 갈림길을 만나지만 그중에서 가장 최근에 만났던 갈림길로 되돌아와야 하므로, 이들을 후입선출 구조의 스택에 저장함
- 그래프는 인접 행렬로 표현함

### 너비 우선 탐색

- BFS는 가까운 정점부터 꼼꼼하게 살피고 먼 정점을 찾아가는 전략을 사용함
- 즉, 거리가 0인 시작 정점으로부터 거리가 1인 모든 정점을 방문하고, 거리가 2인 정점들, 거리가 3인 정점들의 순서로 방문을 진행하는데, 이것은 이진 트리의 레벨 순회와 동작이 비슷함
- BFS에서는 가까운 거리에 있는 정점들을 차례로 저장하고, 들어간 순서대로 꺼낼 수 있는 자료구조가 필요한데, 물론 큐를 사용함
- BFS는 큐에서 정점을 꺼낼 때마다 아직 방문하지 않은 모든 인접 정점들을 방문하고 큐에 삽입함
- 이러한 탐색 과정은 큐가 공백 상태가 될 때까지 계속됨
- 맨 처음 큐에는 시작 정점만이 저장됨
- 그래프는 인접 리스트로 표현함

## 신장 트리

- 신장 트리(spanning tree)는 그래프 내 모든 정점을 포함하는 트리를 말함
- 즉, 그래프의 정점은 모두 포함하고, 간선은 일부만을 포함해 트리의 형태(사이클이 없어야 함)를 이루어야 함
- 만약 그래프의 정점 수가 $n$이라면 신장 트리는 정확히 $n-1$개의 간선으로 모든 정점을 연결해야 함
- 깊이 우선이나 너비 우선 탐색 도중에 사용된 간선들만 모으면 신장 트리가 만들어짐

## 최소 비용 신장 트리

- 최소 신장 트리(MST) 또는 최소 비용 신장 트리는 이처럼 가중치 그래프의 여러 신장 트리 중에서 간선의 가중치 합이 최소인 것을 말함

### 프림(Prim) 알고리즘

- 프림은 하나의 정점에서부터 시작하여 최소 신장 트리(MST)를 단계적으로 확장해나가는 방법을 사용함
- 처음에는 MST에 시작 정점만 포함되고, 다음부터 현재까지 만들어진 MST에 인접한 정점 중에서 간선의 가중치가 가장 작은(최소 간선) 정점을 선택하여 MST를 확장함
- 이 과정은 모든 정점이 삽입될 때까지 계속 됨

**프림의 최소 신장 트리 알고리즘(자연어)**

1. Prim()
2. 그래프에서 시작 정점을 선택하여 초기 트리(MST)를 만듦
3. MST와 인접한 정점 중 간선의 가중치가 가장 작은 정점 v를 선택함
4. v와 이때의 간선을 MST에 추가함
5. 아직 모든 정점이 삽입되지 않았으면 3행으로 이동함

- 이 알고리즘에서 3행이 문제인데, MST에 인접한 정점 중에서 간선의 가중치가 가장 작은 정점을 찾는다는 개념이 애매해 보임
- 이를 위해 정점들의 상태를 저장할 배열을 2개 사용함
- 배열의 크기는 정점 수와 같음

- selected[]: 정점이 MST에 포함되었는지를 기록함
- selected[v]가 True라면 v가 MST에 포함된 것임
- 맨 처음에는 MST가 공백 트리이므로 배열의 모든 요소가 False가 되고, 단계마다 선택되는 정점이 True로 변경됨

- dist[]: 현재까지 구성된 MST와 정점 사이의 최단 거리를 저장함
- 처음에는 시작 정점의 값만 0이고, 나머지는 모두 무한대임
- 새로운 정점 u가 MST에 추가되면 u에 인접한 정점 v의 최단 거리 dist[v]가 더 짧아질 수 있음
- 만약 간선(u, v)의 가중치가 기존의 dist[v]보다 적으면 dist[v]를 (u, v)의 가중치로 변경하는 것임

- 이제 알고리즘 3행을 확실하게 처리할 수 있음
- 아직 선택되지 않은(`selected`가 False인) 정점 중에서 `dist`가 최소인 것을 찾으면 됨

### 크루스칼(Kruskal) 알고리즘

- 크루스칼은 **간선**을 가중치가 작은 순서대로 하나씩 검사하면서, 사이클을 만들지 않는 간선만 골라 MST를 완성하는 방법임
- 프림이 정점을 중심으로 트리를 키워 나간다면, 크루스칼은 처음에는 정점만 따로 떨어져 있다고 보고 간선을 이어 붙이는 방식임
- 어떤 간선 $(u, v)$를 추가했을 때 $u$와 $v$가 이미 같은 연결 요소(같은 부분 트리)에 속해 있으면 그 간선은 사이클을 만들므로 버림
- 반대로 서로 다른 연결 요소에 있으면 그 간선을 MST에 넣고, 두 요소를 하나로 합침
- 정점이 $n$개이면 MST는 정확히 $n-1$개의 간선으로 끝나므로, 그만큼 간선을 고르면 알고리즘이 종료됨

**크루스칼의 최소 신장 트리 알고리즘(자연어)**

1. Kruskal()
2. 그래프의 모든 간선을 가중치의 오름차순으로 정렬함
3. MST를 공백 상태(간선 없음)로 시작함. 처음에는 각 정점이 서로 다른 집합(연결 요소)에 있다고 생각함
4. 정렬된 간선 목록에서 아직 처리하지 않은 가장 작은 가중치의 간선 $(u, v)$를 하나 꺼냄
5. $u$와 $v$가 이미 같은 집합에 속해 있으면(이 간선을 넣으면 사이클이 생기면) 이 간선은 건너뜀
6. 그렇지 않으면 $(u, v)$를 MST에 추가하고, $u$와 $v$가 속했던 두 집합을 하나로 합침
7. MST에 포함된 간선이 $n-1$개가 될 때까지 4행부터 반복함
