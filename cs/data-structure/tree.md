# 트리

## 트리란?

- 트리 구조는 계층적인 관계를 가진 자료의 표현에 매우 유용하게 사용됨 (예: 회사의 조직도)
- 운영체제의 파일 시스템이 트리 구조로 이루어져 있음
- 효율적인 탐색을 위해서도 트리가 사용됨
- 우선순위 큐를 효율적으로 구현하기 위해 트리가 사용됨
- 의사결정 구조를 표현하기 위한 중요한 방법으로 결정 트리(decision tree)가 사용됨

### 트리 관련 용어

- 노드(node): 트리에서 각각의 요소들
- 트리는 하나 이상의 노드들로 이루어짐
- 간선 / 에지(edge): 노드와 노드의 연결 관계
- 루트(root) 노드: 계층적인 구조에서 가장 높은 곳에서 있는 노드

- 부모(parent) 노드: 간선으로 직접 연결된 노드 중에 상위 노드
- 자식(child) 노드: 간선으로 직접 연결된 노드 중에 하위 노드
- 형제(sibling) 노드: 같은 부모 노드를 가진 노드
- 조상(ancestor) 노드: 어떤 노드에서 루트 노드까지의 경로상에 있는 모든 노드
- 자손(descendent) 노드: 어떤 노드 하위에 연결된 모든 노드
- 단말(terminal, leaf) 노드: 자식 노드가 없는 노드 / 자식이 있으면 비단말 노드
- 노드의 차수(degree): 노드가 가지고 있는 자식의 수 / 단말 노드는 항상 0
- 트리의 차수: 트리에 포함된 모든 노드의 차수 중에서 가장 큰 수
- 레벨(level): 트리의 각층에 번호를 매기는 것 / 루트 노드의 레벨은 1이고, 한 층씩 내려갈 수록 레벨은 1씩 증가
- 트리의 높이(height): 트리가 가지고 있는 최대 레벨

- 트리의 모든 노드는 자신을 루트로 하는 하나의 서브 트리를 대표함
- 따라서 트리는 순환적으로 정의되는 자료구조라고 볼 수 있고, 순환 호출을 사용하는 순환 알고리즘이 흔히 사용됨

### 트리 표현 방법

1. 중첩된 집합

- 트리가 서브 트리의 집합이라는 것을 잘 나타낼 수 있음

2. 트리의 루트와 서브 트리를 중첩된 괄호로 묶어 표현

- 트리를 하나의 문자열처럼 표현할 수 있다는 장점이 있으나 이것만으로 트리의 구조를 직관적으로 이해하기에는 다소 어려움이 있음

3. 들여쓰기

- 자료가 계층적인 구조를 갖는 것을 잘 나타냄
- 윈도 탐색기에서 폴더와 파일을 나타내기 위해 흔히 사용됨

### 일반 트리의 표현

- 일반 트리(general tree): 자식의 개수에 제한이 없는 트리

1. N-링크 표현

- 차수가 N인 노드가 N개의 링크를 갖도록 허용하는 것임
- 이 방법은 간단해 보이지만 노드마다 링크의 개수가 다르다는 문제가 있음
- 이를 위해 노드별로 링크를 관리하기 위해 연결 리스트를 사용할 수 있지만, 트리의 표현과 관리가 복잡해 질 수밖에 없음

2. 왼쪽 자식-오른쪽 형제 표현

- 노드의 형태를 좀 더 단순화해서, 노드가 N개가 아니라 두 개의 링크만을 갖도록 하는 방법
- 링크의 용도를 정해서 두 개의 링크만으로 많은 자식을 갖는 트리를 표현할 수 있음
- 하나의 링크는 왼쪽 자식(첫 번째 자식, child)을 가리키고, 다른 하나는 오른쪽 형제(다음 형제, sibling)을 가리키기 위해 사용하는 것임
- 이 방법은 비교적 단순한 형태의 노드를 이용해 임의의 일반 트리를 표현할 수 있음
- 그러나 표현이 복잡하고 특히 루트에서 마지막 레벨까지 찾아가는 과정에서 필요 없이 많은 노드를 거쳐야 하는 문제가 있음

- 다행히 실제로는 좀 더 단순한 형태의 트리를 많이 사용하는데, 트리 노드의 자식 수에 제한을 두는 것임

## 이진 트리

- 이진 트리(binary tree)는 모든 노드가 최대 2개의 자식만을 가질 수 있는 트리임
- 즉, 모든 노드의 차수가 2 이하로 제한된 트리임
- 이때 자식 사이에도 순서가 존재하는데, 왼쪽 자식과 오른쪽 자식은 반드시 구별되어야 함
- 이 트리는 무제한의 자식을 허용하는 일반 트리에 비해 계층적인 관계를 가진 모든 자료를 표현하기 부족함
- 그러나 이러한 특별한 구조를 이용해 여러 훌륭한 알고리즘들이 개발되어 있음
- 이진 탐색 트리(binary search tree): 아주 빠른 자료의 탐색이 가능함
- 힙 트리(heap tree): 우선순위 큐를 효과적으로 구현함
- 수식 트리(expression tree): 수식을 트리 형태로 표현하여 계산함

### 이진 트리의 종류

- 이진 트리는 레벨과 노드 수의 관계에 따라 포화 이진 트리와 완전 이진 트리를 정의할 수 있음
- 또한 서브 트리의 높이에 '균형'의 개념을 적용하면 균형 이진 트리를 정의할 수 있음

**포화 이진 트리(full binary tree)**

- 트리의 각 레벨에 노드가 꽉 차있는 이진 트리
- 노드가 꽉 차 있으니까 트리의 높이를 알면 전체 노드의 수를 쉽계 계산할 수 있음
- 높이가 k인 포화 이진 트리의 노드의 수: 2^k - 1
- 포화 이진 트리는 각 노드에 순서대로 번호를 붙일 수 있음
- 위에서 아래로 내려오면서 왼쪽에서 오른쪽으로 순서대로 번호를 붙이면 되는데, 이 번호는 항상 일정함

**완전 이진 트리(complete binary tree)**

- 높이가 k인 트리에서 레벨 1부터 k-1까지는 노드가 모두 채워쟈 있고, 마지막 레벨 k에서는 왼쪽부터 오른쪽으로 노드가 순서대로 채워져 있는 이진 트리
- 마지막 레벨에서는 노드가 꽉 차 있지 않아도 되지만 중간에 빈 곳이 있으면 안 됨
- 따라서 포화 이진 트리는 완전 이진 트리이지만 그 역은 항상 성립하지 않음
- 힙(heap)은 완전 이진 트리의 대표적인 예임

**균형 이진 트리(balanced binary tree)**

- 이진 트리는 좌우 서브 트리를 구분하기 때문에 '균형'의 개념을 적용할 수 있음
- 균형 이진 트리 또는 높이균형 이진 트리(height-balanced binary tree)는 모든 노드에서 좌우 서브 트리의 높이 차이가 1 이하인 트리를 말함
- 경사 트리(경사 이진 트리): 다른 노드는 모두 균형 잡혀 있지만, 노드 C는 좌우 서브 트리의 높이 차이가 2임

### 이진 트리의 표현 방법

- 이진 트리도 배열 구조와 연결된 구조롤 표현할 수 있음

**배열 구조 표현**

- 이진 트리를 포화 이진 트리의 일부라고 생각하면 배열 구조로 트리를 표현할 수 있음
- 포화 이진 트리에서 노드에 번호를 붙이는 것을 그대로 배열의 인덱스로 사용하는 것임

1. 트리의 높이를 구해 배열을 할당함 예를 들어 높이가 k이면 길이가 2^k -1인 배열이 필요함
2. 포화 이진 트리의 번호를 인덱스로 사용하여 배열에 노드들을 저장함

- 보통 루트 노드의 번호를 1로 하는데, 계산의 편의를 위해 인덱스 0은 사용하지 않음
- 이 방법은 포화 이진 트리나 완전 이진 트리에 가장 적잡하지만, 일반 이진 트리도 표현할 수 있음
- 다만 심한 경사 트리의 경우, 배열 항목 사이에 사용하지 않는 빈칸이 많이 발생하여 메모리의 낭비가 심해질 수 있음
- 이 표현법에서는 어떤 노드의 인덱스를 알면 부모 노드나 자식 노드의 인덱스를 손쉽게 계산할 수 있음
  - 노드 i의 부모 노드 인덱스 = i//2
  - 노드 i의 왼쪽 자식 노드 인덱스 = 2i
  - 노드 i의 오른쪽 자식 노드 인덱스 = 2i + 1
- 이러한 배열 표현법은 간단하지만 경사 트리에서 메모리를 낭비할 수 있으며, 배열의 크기에 따라 표현 가능한 트리의 높이가 제한되는 단점이 있음

**연결된 구조 표현: 링크 표현법**

- 연결된 구조로도 이진 트리를 나타낼 수 있음
- 이진 트리를 위한 노드는 두 개의 링크가 필요한데, 이들은 각각 왼쪽(left)과 오른쪽(right) 자식 노드를 가리킴
- 이때 좌우 링크는 반드시 구별되어야 함

- 부모와 자식의 관계를 나타내기 위해서는 간선을 화살표로 나타내는 것이 더 정확하지만, 화살표가 없이 연결된 경우에는 위쪽이 부모 노드, 아래쪽이 자식 노드임
- 이진 트리를 위한 노드 클래스를 BTNode라고 하면, 다음과 같이 표현할 수 있음
- 생성자에서 데이터 필드와 왼쪽 자식 및 오른쪽 자식에 대한 링크를 생성하고 초기화함

```python
class BTNode:
    def __init__ (self, elem, left=Node, right=Node):
        self.data = elem
        self.left = left # 왼쪽 자식을 위한 링크
        self.right = right # 오른쪽 자식을 위한 링크
```

## 이진 트리의 연산

- 이진 트리는 루트와 두 개의 서브 트리로 이루어지는데, 서브 트리도 모두 이진 트리여야 한다는 순환적인 조건이 있음
- 이러한 순환적인 특징에 의해 이진 트리의 연산에는 순환 알고리즘이 흔히 사용됨
- 따라서 먼저 순환 호출을 복습하는 것이 좋음

### 이진 트리의 표준 순회

- 트리를 순회한다는 것은 트리의 모든 노드를 한 번씩 방문하는 것을 말함
- 예를 들어, 트리의 모든 노드를 한 번씩 화면에 출력하기 위해서도 순회가 필요한데, 순회는 모든 자료구조에서 기본적인 연산의 하나임
- 일단 선형 자료구조에서는 항목들이 일렬로 저장되어 있어서 순회가 매우 단순함
- 순서대로 방문하면 되기 때문임
- 그렇지만 트리에서는 자료가 일렬로 나열되어 있지 않기 때문에 여러 가지 '순서'로 노드를 방문할 수 있고, 따라서 순회 방법이 다양함
- 기본적인 순회 방법은 전위, 중위, 후위로 나눌 수 있는데, 이를 이진 트리의 표준 순회라고 함
- 이들은 루트와 왼쪽 서브 트리, 오른쪽 서브 트리를 각각 어떤 순서로 방문하느냐에 따라 구분됨
- 루트를 방문하는 작업을 V, 왼쪽과 오른쪽 서브 트리를 방문하는 작업을 각각 L과 R이라 하면, 이들을 처리하는 순서에 따라 세 가지로 나눌 수 있음

  1. 전위순회: VLR
  2. 중위순회: LVR
  3. 후위순회: LRV

- 루트 노드의 방문 작업 V는 간단함
- 서브 트리의 방문 작업 L과 R을 처리하는 데 동일한 순회 방법이 적용되어야 함
- 트리에서는 전체 트리나 서브 트리나 기본 구조가 완전히 같으므로 전체 트리에 사용된 알고리즘은 똑같이 서브 트리에도 적용할 수 있음
- 따라서 순환 기법(재귀 호출)이 흔히 사용됨
- 물론 순환이 진행됨에 따라 문제의 크기는 작아짐

**1. 전위 순회(preorder)**

1. 루트를 먼저 방문(V)하고 다음에
2. 왼쪽 서브 트리를 방문(L)하고, 방문이 끝나면 마지막으로
3. 오른쪽 서브 트리를 방문(R)하는 방식

```python
def preorder(n): # 전위순회 함수
    if n is not None:
        print(n.data, end= ' ') # 노드를 방문해 처리할 연산들의 위치, 여기서는 노드의 데이터를 단순히 화면에 출력함
        preorder(n.left) # 왼쪽 서브 트리
        preorder(n.right) # 오른쪽 서브 트리
```

- 전위순회를 사용하면 이진 트리를 중첩된 괄호로 표현할 수 있음
- 루트부터 시작해 순환호출을 할 때마다 괄호를 한 겹씩 두르면 됨
- 전위순회 결과: A B D E C F
- 중첩된 괄호 표현: ( A ( B ( D ) ( E ) ) ( C ( F ) ) )

**2. 중위순회(inorder)**

1. 왼쪽 서브 트리부터 시작해서
2. 루트를 거쳐
3. 오른쪽 서브 트리 순으로 방문함

```python
def inorder(n):
    if n is not None:
        inorder(n.left) # 왼쪽 서브 트리 처리
        print(n.data, end= ' ') # 노드에서 처리할 연산들의 위치
        inorder(n.right) # 오른쪽 서브 트리 처리
```

**3. 후위순회(postorder)**

1. 왼쪽 서브 트리
2. 오른쪽 서브 트리
3. 루트 순으로 방문함

```python
def postorder(n):
    if n is not None:
        postorder(n.left)
        postorder(n.right)
        print(n.data, end= ' ') # 노드에서 처리할 연산들의 위치
```

**순회 방법의 선택**

- 순서는 중요하지 않고 노드를 전부 방문하기만 하면 된다면 어떤 방법도 상관 없음
- 예를 들어, 모든 노드 값을 순서와 상관없이 출력하면 되는 문제에서는 순회 방법이 중요하지 않음

- 자식을 먼저 처리해야 부모를 처리할 수 있다면 당연히 후위순회를 사용해야 함
- 예를 들어, 컴퓨터에서 어떤 폴더의 용량은 하위 폴더들의 용량을 알아야만 계산할 수 있기에 후위순회를 사용해야 함

- 부모가 처리되어야 자식을 처리할 수 있다면 전위순회를 사용해야 함
- 예를 들어, 모든 노드에서 자신의 레벨을 계산하는 경우, 전위순회를 사용해야 함
- 루트의 레벨이 1이고, 나머지 모든 노드는 부모의 레벨보다 1이 크기 때문에, 부모의 레벨이 결정되어야 자식의 레벨을 결정할 수 있음

### 레벨 순회

- 레벨 순회(level order)는 레벨 순으로 노드를 방문함
- 루트의 레벨이 1이고, 아래로 내려갈수록 레벨이 증가하므로 위에서 아래로 방문하고, 같은 레벨에서는 왼쪽에서 오른쪽으로 방문함

- 이러한 순서는 큐를 이용하여 간단하여 구현할 수 있음
- 큐를 노드에 넣고 빼는 과정을 반복하는 것임
- 맨 처음에는 공백 상태의 큐에 루트를 넣음
- 다음부터는 큐에서 노드 하나를 꺼내 방문하고 그 노드의 자식들을 삽입하는 과정을 반복함
- 삽입에도 순서가 있는데, 왼쪽 자식을 먼저 넣고 오른쪽 자식을 다음에 넣음
- 물론 자식이 없으면 삽입하지 않음
- 이 과정은 큐가 공백 상태가 될 때까지 반복됨

- 이러한 레벨 순회는 순환을 사용하지 않음

```python
def levelorder(root):
    queue = ArrayQueue() # 큐 객체 초기화
    queue.enqueue(root) # 최초에 루트 노드만 들어있음
    while not queue.isEmpty(): # 큐가 공백 상태가 아닌 동안
        # 큐에서 하나의 노드를 꺼내고, 이 노드가 None이 아니라면 처리 / 마지막으로 이 노드의 왼쪽과 오른쪽 자식 노드를 큐에 삽입
        n  = queue.dequeue()
        if n is not None:
            print(n.data, end = ' ')
            queue.enqueue(n.left)
            queue.enqueue(n.right)
```

### 이진 트리의 연산들

**전체 노드의 수 구하기**

- 이진 트리에서 어떤 문제를 해결하기 위해서는 일단은 순환을 떠올려 보는 것이 좋음
- 노드의 수를 구하는 경우도 마찬가지임
- 이진 트리의 노드 개수는 왼쪽 서브 트리의 노드의 수와 오른쪽 서브 트리의 노드 수의 합에 1(루트 노드)을 더하면 됨

```python
def count_node(n):
    if n is None: # n이 None이면 공백 트리 -> 0을 반환
        return 0
    else:
        return count_node(n.left) + count_node(n.right) + 1 # 좌우 서브 트리의 노드 수의 합 + 1 (순환 이용)
```

**트리의 높이 구하기**

- 높이 계산에도 순환 아이디어를 사용할 수 있음
- 이진 트리의 높이는 왼쪽 서브 트리의 높이와 오른쪽 서브 트리의 높이 중에서 큰 값에 1을 더한 값임

```python
def calc_height(n):
    if n is None: # 공백 트리 -> 0을 반환
        return 0
    hLeft = calc_height(n.left) # hLeft: L의 높이
    hRight = calc_height(n.right) # hRight: R의 높이
    if (hLeft > hRight): # 더 큰 값에 1을 더해 반환
        return hLeft + 1
    else: return hRight + 1
```
