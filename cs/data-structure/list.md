# 리스트

### 리스트는 가장 자유로운 선형 자료구조임

- 리스트(list) 또는 선형 리스트(linear list)는 자료들이 차례대로 나열된 선형 자료구조임
- 이때 각 자료는 순서 또는 위치(position)를 가짐
  - 삽입과 삭제가 임의의 위치에서 가능함
  - 중간에 비어있는 위치는 없어야 함
- 리스트에서는 어떤 위치에서도 새로운 요소를 삽입할 수 있음
- 이때 중요한 것은 어느 위치에 요소를 삽입하려면 이후 모든 자료가 한 칸씩 뒤로 밀린다는 것임
- 삭제도 마찬가지임
- 어느 위치의 요소를 삭제하면 이후의 모든 요소의 위치가 변경됨
- 가장 활용이 자유로운 선형 자료구조임
- 자유로운 만큼 고려해야 할 사항들도 많고, 코드도 복잡해질 것임
- 리스트는 집합(Set)과 유사한 점이 많지만 중요한 차이가 있음
  - 리스트에서는 순서가 있지만 집합은 원소 사이에 순서가 없음
  - 또한 집합은 원소의 중복을 허용하지 않음
  - 특히 집합은 원소 사이에 순서의 개념이 없으므로 선형 자료구조라 볼 수 없음

### 리스트의 연산

```python
insert(pos, e): pos 위치에 새로운 요소 e를 삽입
delete(pos): pos 위치에 있는 요소를 꺼내서 반환
getEmpty(pos): pos 위치에 있는 요소를 삭제하지 않고 반환
isEmpty(): 리스트가 비어 있으면 True를, 아니면 False를 반환
isFull(): 리스트가 가득 차 있으면 True를, 아니면 False를 반환
size(): 리스트에 들어 있는 전체 요소의 수를 반환
```

### 추가 연산

```python
append(e): 맨 뒤에 새로운 요소를 삽입
pop(): 맨 뒤의 요소를 삭제
fine(e): 요소 e의 위치를 찾음
replace(pos, e): pos번째 요소를 변경
display(): 리스트를 화면에 보기 좋게 출력
```

### 배열 구조와 연결된 구조

- 리스트도 배열 구조와 연결된 구조로 구현할 수 있음
- 배열 구조에서는 모든 요소는 중간에 빈자리가 없이 반드시 메모리의 연속된 공간에 저장되어야 함
- 요소들이 연속된 공간에 있으면 원하는 위치의 요소를 빠르게 참조하고 관리할 수 있음
- 이에 비해 연결된 구조에서는 요소들을 메모리의 한군데 모아두는 것을 포기함
- 즉, 요소들이 메모리의 여기저기에 흩어져서 저장되는 것임
- 흩어진 요소들은 링크(link)를 이용해 순서대로 관리함
- 즉, 요소들이 다른 요소를 가리키는 하나 이상의 링크를 갖도록 하여 전체를 순서대로 연결해 관리하는 것임
- 이처럼 메모리에 흩어져 있는 요소들을 링크로 연결해 하나로 관리하는 것을 연결된 구조(linked structure)라고 함
- 특히 자료들을 링크를 통해 일렬로 나열할 수 있는 연결된 구조를 연결 리스트(linked list)라고 부름
- 연결된 구조에서 하나의 상자를 노드(node)라고 부르는데, 데이터와 함께 링크를 가짐
- 배열 구조와는 달리 하나의 요소를 표현하기 위해 링크가 추가로 필요한 것에 유의해야 함
- 연결된 구조에서는 이러한 링크의 수를 여러 개로 늘리면 리스트와 같은 선형 자료구조뿐만 아니라 트리나 그래프와 같이 훨씬 복잡한 구조들도 효율적으로 표현할 수 있음

### 배열 구조의 리스트와 연결 리스트의 비교

**리스트 요소들에 대한 접근**

- 배열 구조는 모든 요소의 크기가 같고 연속된 메모리 공간에 있음
- 또한 배열의 시작 주소와 요소 하나의 크기를 알고 있음
- 예를 들어 A[3]의 주소는 한 요소의 크기에 원하는 위치(3)를 곱해 시작 주소에 더해주면 바로 계산됨
- 계산하는 데 걸리는 시간은 k와 상관없음
- 즉, k가 아무리 크더라도 곱셈과 덧셈을 각각 한 번씩만 사용하면 위치가 계산됨
- 반대로 연결된 구조는 k번째 요소를 찾기 위해 큰 노력이 필요함
- 연결된 구조에서는 시작 노드의 주소만 알고 있는데, k번째 요소를 찾으려면 어쩔 수 없이 시작 노드에서부터 k-1번 링크를 따라 이동해야 함
- 만약 k가 작은 값이라면 몇 번만 이동하면 되겠지만 k가 큰 값이라면 많은 이동을 거쳐야 드디어 원하는 요소에 도달할 수 있음
- 리스트에서 원하는 요소에 접근하는 연산은 배열 구조가 훨씬 유리함

**리스트의 용량**

- 기본적으로 배열은 용량이 고정됨
- 고정된 용량은 중간에 부족하다고 늘리거나 필요 없다고 줄이기 어려움
- 리스트를 위해 배열을 사용할 때, 배열을 무턱대고 너무 크게 할당해 놓으면 메모리 낭비가 심할 것임
- 그렇다고 너무 적게 할당하면 빨리 포화 상태가 되어 새로운 요소를 넣지 못하는 상황이 발생함
- 이에 비해 연결된 구조는 용량이 고정되어 있지 않음
- 필요할 때만 필요한 크기만큼 새로 할당해서 사용하는 것임
- 따라서 메모리를 효율적으로 사용할 수 있음
- 또한 컴퓨터에 메모리가 남아있는 한 계속해서 자료를 넣을 수 있음
- 따라서 가득 찬 상태, 즉 포화 상태가 될 수 없음

**리스트의 삽입 연산**

- 배열 구조에서는 중간에 자료를 삽입하려면 그 위치 이후의 모든 요소를 한 칸씩 뒤로 밀어야 함
- 이때 이동은 맨 뒤의 요소부터 앞으로 가면서 처리해야 함
- 이처럼 배열 구조의 삽입은 많은 요소의 이동이 필요함
- 연결된 구조에서는 삽입할 위치를 알고 있다면 효율적인 삽입이 가능함
- 삽입할 위치 바로 전 노드의 링크와 새로운 요소의 링크만 수정하면 됨
- 그 외의 다른 노드들은 전혀 영향을 받지 않음
- 결국 삽입할 위치 바로 앞의 노드를 알고 있다면 연결된 구조의 삽입이 훨씬 효율적임

**리스트의 삭제 연산**

- 배열 구조에서는 중간에 있는 자료를 삭제하려면 이후의 모든 자료를 앞으로 당겨 빈 곳이 생기지 않도록 해야 함
- 연결된 구조에서는 삭제할 노드 바로 앞 노드의 링크만 수정하면 되고, 다른 노드들은 수정할 필요가 없음
- 결국 삭제할 위치 바로 앞의 노드를 알고 있다면 연결된 구조의 삭제가 훨씬 효율적임

### 연결 리스트의 구조

**노드(node)**

- 연결 리스트에서 하나의 노드(node)는 하나의 데이터(data)와 함께 하나 이상의 링크(link)를 가짐
- 이때 데이터는 배열 구조에서의 요소를 말하는데, 리스트에 저장하고 싶은 자료에 해당함
- 링크는 다른 노드를 가리키는(다른 노드의 주소를 저장하는) 변수임
- 배열 구조에 비해 연결된 구조에서는 링크를 위한 추가 공간이 필요함
- 그러나 대부분은 데이터가 훨씬 크므로 링크를 위한 공간은 무시할 수 있음

**헤드 포인터(head pointer)**

- 연결 리스트는 시작 노드만 알면 링크로 매달려 있는 모든 노드에 순차적으로 접근할 수 있음
- 이 노드를 보통 머리 노드(head node)라고 부름
- 머리 노드의 주소를 저장하는 변수를 헤드 포인터(head pointer)라고 함
  - 연결 리스트의 가장 중요한 정보임
- 마지막 노드를 보통 꼬리 노드(tail node)라고 부르는데, 꼬리 노드의 링크를 처리하는 방법에 따라 단순 연결과 원형 연결로 구분됨

### 연결 리스트의 종류

**단순 연결 리스트(singly linked list)**

- 하나의 방향으로만 연결된 리스트를 말함
- 노드는 하나의 링크를 갖는데, 다음 노드의 주소가 저장됨
- 꼬리 노드의 링크는 마지막 노드라는 것을 나타내기 위해 None을 갖기로 약속되어 있음

**원형 연결 리스트(circular linked list)**

- 꼬리 노드의 링크를 약간 다르게 사용하면 원형으로 연결된 구조를 만들 수 있음
- 꼬리 노드의 링크가 None이 아니라 다시 머리 노드를 가리키도록 하는 것임
- 이러한 원형 연결 구조에서는 어떤 노드에서 시작해도 다른 모든 노드를 찾아갈 수 있음
- 노드들을 순서대로 방문할 때 종료 조건 처리에 매우 조심해야 함

**이중 연결 리스트(doubly linked list)**

- 하나의 노드가 이전 노드와 다음 노드의 링크를 모두 갖도록 설계되어 있음
- 두 개의 링크를 갖는데, 하나는 이전 노드(previous node)를, 다른 하나는 다음 노드(next node)를 가리키도록 하는 것임
- 이전 노드를 위한 링크가 있으면 어떤 노드에서 이전 노드를 바로 찾아갈 수 있다는 장점이 있음
- 편리한 만큼 링크를 이중으로 정확히 유지해야 하므로 코드가 복잡해짐
