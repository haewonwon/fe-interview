# 탐욕적 기법

- 억지 기법과 비슷하게 탐욕적 기법(greedy method)도 매우 단순하고 직관적인 전략을 사용함
- 다만 모든 경우를 고려해 보고 가장 좋은 답을 찾는 것이 아니라 '그 순간에 최적'이라고 생각되는 답을 선택함
- 즉, 억지 기법과 달리 멀리 내다보지 않고 앞에 있는 가까운 것들만 보고 결정하기 때문에 "근시안"적인 알고리즘이라 할 수 있음

- 순간에 최적이라고 판단했던 선택들을 모아 만든 최종적인 답이 궁극적으로 최적, 즉 최적해가 **되지 않을 수 있음**
- 따라서 탐욕적 알고리즘은 최적의 해답을 주는지를 반드시 검증해야 하는데, 실제로 많은 경우 최적해를 찾지 못함
- 따라서 탐욕적 기법이 사용될 수 있는 경우는 두 가지로 제한됨

  1. 탐욕적 기법으로도 최적해를 구할 수 있다면 당연히 이것을 사용하는 것이 좋음

  - 다른 전략들보다 시간적으로 훨씬 효율적이기 때문임
  - MST를 위한 프림 알고리즘, 최단경로거리를 구하는 다익스트라 알고리즘 등

  2. 시간이나 공간적인 제약으로 최적해가 현실적으로 불가능한 경우에 사용할 수 있음

  - 탐욕적 기법이 최적해보다는 못하지만 임의로 선택한 답보다는 훨씬 좋기 때문임
  - 예를 들어, 분기 한정 기법에서 좋은 한계값을 구하기 위해 탐욕적 기법이 사용될 수 있음

- 탐욕적 기법은 결정 순간에 가능한 해 중에 지역적으로 최적인 것을 선택하고, 이러한 선택은 이후의 단계에서 다시 변경될 수 없음

### 거스름돈 동전 최적화

- 액면가가 서로 다른 $m$가지의 동전이 있습니다. 거스름돈으로 $V$원을 동전으로만 돌려주어야 한다면 최소 몇 개의 동전이 필요한지를 구하세요. 단, 모든 동전은 무한히 사용할 수 있고, 액수가 큰 것부터 내림차순으로 순서대로 정렬되어 있습니다.
- 우리나라에서는 {500원, 100원, 50원, 10원, 5원, 1원}의 6가지 동전이 사용되고 있음

- 거스름돈 620원: 500원 + 100원 + 10원 × 2 → 동전 4개
- 거스름돈 345원: 100원 × 3 + 10원 × 4 + 5원 → 동전 8개
- 거스름돈 572원: 500원 + 50원 + 10원 × 2 + 1원 × 2 → 동전 6개

- 남은 거스름돈에 대해 액면가가 가장 높은 동전부터 탐욕적으로 최대한 사용하면서 거스름돈을 맞추면 동전 개수를 최소로 만들 수 있음

**최적해를 구할까?**

- 그렇게 간단하지는 않음
- 만약, 우리나라에서 기념주화로 60원짜리를 추가로 만들었다면, 거스름돈 620원을 만들기 위해 500원짜리 하나와 60원짜리 두 개, 즉 3개의 동전으로 거스름돈을 만들 수 있음
- 그런데 탐욕적으로 액면가가 높은 것부터 사용한다면,
- 620원이 남았으므로 500원짜리 한 개를 사용함 (잔돈 120원)
- 액면가가 가장 높은 100원짜리를 한 개 사용함 (잔돈 20원)
- 액면가가 가장 높은 10원짜리를 두 개 사용해 나머지를 맞춤
- 따라서 총 4개의 동전이 필요하고, 이것은 최적해(3개)가 아님
- 결국 동전 최소화 문제는 탐욕적 기법으로 최적해를 구하지 못함

- 억지 기법을 적용할 수 있지만, 지수적인 시간 복잡도를 갖게 되므로 매우 비효율적임
- 동적 계획법이 좋은 선택인데, $O(mV)$의 시간 복잡도와 $O(V)$의 공간 복잡도를 갖게 됨
- 이때 $m$은 서로 다른 액면가의 종류 수이고, $V$는 거스름돈 액수임

- 이 문제에 대해 탐욕적 전략은 최적해를 보장하지 못하지만 다음과 같은 동전 체계를 가진다면 이야기는 달라짐
- 주어진 액면가 중에서 어떤 두 개를 고르더라도, 큰 액면가가 작은 액면가로 **나누어떨어지는** 동전 체계라면 탐욕적 기법으로도 최적해를 보장함
- 작은 액면가를 여러 개 모으면 반드시 큰 액면가를 만들 수 있기 때문임

### 분할 가능한 배낭 채우기

- 이 문제에 대한 두 가지 '탐욕'을 생각해 보면,

1. 탐욕 1: 무게와 상관없이 가장 비싼 물건부터 넣어보는 방법
2. 탐욕 2: 단위 무게당 가격이 가장 높은 물건부터 넣어보는 방법

**0-1 배낭 문제는 탐욕적 기법으로 최적해를 구하지 못함**

- 불행히도 이 방법들은 모두 최적해를 구하지 못함
- 최적해가 아닌 상황을 간단히 제시할 수 있기 때문임
- 세 개의 물건 A=(12kg, 120만원), B=(10kg, 80만원), C=(8kg, 60만원)가 있고, 배낭의 용량이 18kg인 경우, 최적해는 B와 C를 넣는 경우이고, 최대 가치는 140만원임

- 탐욕 1: 가장 비싼 물건은 A이고, A를 선택하면 용량 때문에 B나 C를 넣을 수 없음 / 배낭 가치는 120만원이 됨 / 최적해 140만원보다 작음
- 탐욕 2: 단위 무게당 가장 비싼 물건도 A임 / A는 kg당 10만원의 가치가 있는데, B나 C는 이보다 가치가 낮음 / 탐욕 1과 같이 최적해가 아님
- 결론적으로 0-1 배낭 채우기 문제는 어떤 '탐욕'을 사용해도 최적해를 만들 수 없음
- 다만 문제를 약간 변형하면 최적해를 구할 수 있음

**분할 가능한 배낭 채우기**

- 만약 물건들이 가루로 되어 있어 일부분만을 배낭에 넣을 수 있다면?
- 이것을 분할 가능한 배낭 채우기(fractional knapsack) 문제라고 함

- 미세한 가루로 된 세 가지 물건, A=(120kg, 120억원), B=(10kg, 80억 원), C=(8kg, 60억원)가 있다고 하면, 이제 물건들은 가루로 되어 있어 마음대로 일부분만 넣을 수 있음
- 이 문제를 탐욕적인 방법으로 해결해 보면, 단위 무게당 가격이 높은 물건부터 넣어야 함
- 0-1 배낭 채우기와는 달리 이 문제에서는 항상 배낭을 최대 용량으로 채울 수 있음
- 물건의 리스트가 입력되면 먼저 무게당 가치의 내림차순으로 정렬함
- 다음을 단가가 가장 높은 것부터 최대한 많이 탐욕적으로 배낭에 넣으면 됨

- 이 알고리즘은 넣을 수 있는 모든 공간을 항상 무게당 가격이 가장 높은 것부터 채우기 때문에 최적해를 확실히 보장함
- 무게당 가치 내림차순 정렬에 $O(n \log n)$, 배낭 채우기에 $O(n)$이 들어 전체 시간 복잡도는 $O(n \log n)$임
- 결론적으로 0-1 배낭 문제는 탐욕적 전략으로는 최적해를 구할 수 없지만, 문제의 조건을 변경한 분할 가능한 배낭 문제는 탐욕적 기법으로도 최적해를 구할 수 있음
