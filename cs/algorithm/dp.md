# DP (동적 프로그래밍)

## 동적 계획법

- 동적 계확법은 분할 정복기법과 많은 부분이 닮았음
- 분할 정복과 같이 주어진 문제를 더 작은 부분 문제로 나눈 다음 답을 계산하고, 이러한 답들로부터 원래 문제의 해답을 구하는 구조는 똑같음
- 다만, 동적 계획법은 부분 문제들의 답을 어딘가에 저장해 놓고 필요할 때 다시 꺼내서 사용하는 전략을 사용함
- 즉, 같은 부분 문제를 다시 풀지 않도록 하는 것임
- 물론 부분 문제의 답을 저장하려면 추가적인 메모리가 필요하고, 따라서 넓게 보면 공간으로 시간을 버는 전략에 포함될 수도 있음

### 피보나치 수열 문제

- 피보나치 수열은 순환 관계로 나타낼 수 있음
- 이러한 순환 관계는 분할 정복을 이용해 해결할 수 있음
- 다만, 심각한 계산 중복 문제가 발생하고, 알고리즘의 시간 복잡도가 O(n^2)이나 되었음
- 동적 계획법은 같은 문제를 다음에 다시 풀어야 할 때는 저장된 해답을 이용함

### 메모이제이션을 이용한 피보나치 수열

- 메모이제이션은 함수의 결과를 저장할 메모리를 미리 준비해서 한 번 계산한 값을 저장해 두었다가 재활용하는 방법을 말함
- 메모이제이션은 분할 정복과 같이 하향식(top-down)으로 문제를 해결함
- 이때 한 번 풀린 문제는 답을 저장하고 재활용하는데, 따라서 모든 부분 문제들은 한 번씩만 풀면 되고, 많은 중복 호출을 피할 수 있음

```python
def fib_dp_mem(n):
    # 처음 푸는 문제라면 풀어서 메모리에 저장함
    if (mem[n] == None):
        if n < 2:
            mem[n] = n
        else:
            mem[n] = fib_dp_mem(n-1) + fib_dp_mem(n-2)
    return mem[n] # 저장된 답을 반환
```

- 이 알고리즘은 전역변수 mem을 사용하고 있는데, mem은 크기가 n+1인 리스트로, mem[k]에는 k번째 피보나치 수가 저장됨
- 함수를 호출하기 전에 mem의 모든 항목은 None(아직 해결되지 않은 문제)으로 반드시 초기화되어 있어야 함
- 모든 부분 문제가 정확히 한 번씩만 처리되고, 이러한 부분 문제의 수는 n개임
- 따라서 시간 복잡도는 O(n)으로, 분할 정복의 O(2^n)과 비교하면 매우 효율적인 알고리즘임
- 물론 이러한 효율을 위해 테이블 mem을 추가로 사용하였는데, 용량은 n에 비례함
- 따라서 공간 복잡도도 O(n)임

### 테이블화를 이용한 피보나치 수열
- 테이블화(타뷸레이션)는 부분 문제의 해를 메모리에 저장한다는 점은 메모이제이션과 같지만, 테이블 항목들을 순서대로 채워나가는 것에 초점을 맞춤
- 즉, 상향식(bottom-up)으로 문제를 해결함
- 가장 작은 부분부터 순서대로 해를 구해 테이블을 채워서 올라감
- 이때, 이미 구한 부분 문제의 결과가 이용되는데, 예를 들어 fib(2)는 이미 저장된 fib(0)과 fib(1)의 결과를 이용해 계산함

```python
def fib_dp_tap(n):
    # 답을 저장할 테이블을 만들고, 알려진 답을 저장함
    # 나머지는 모두 None으로 초기화함
    f = [None] * (n+1)
    f[0] = 0
    f[1] = 1
    # 순서대로(상향식) 문제를 풀고, 결과를 테이블에 저장함
    for i in range(2, n+1):
        f[i] = f[i-1] + f[i-2]
    return f[n]
```

- 이 알고리즘은 순환 호출을 사용하지 않고, 5행에서 n번 반복하는 하나의 루프를 사용하므로 시간 복잡도는 명백히 O(n)임
- 메모이제이션과 같이 O(n)의 추가 메모리를 사용하지만, 테이블을 전역 변수로 선언할 필요는 없음

### 동적 계획법을 이용한 문제 해결 전략

1. 분할 정복 전략을 적용할 수 있어야 함
2. 피보나치 수열처럼 중복된 문제를 반복해서 해결하는 경우여야 함
    - 이진 탐색이나 병합 정렬 등에서는 중복된 부분 문제가 발생하지 않으므로 동적 계획법이 적합하지 않음
3. 최적 부분 구조 특성이 있어야 함
    - 부분 문제의 최적해를 이용하면 전체 문제를 해결할 수 있는 구조를 의미함
